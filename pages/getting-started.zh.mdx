# 快速开始

开始使用 Redefine 主题。

<Callout type="info">
  Hexo 版本需要大于 5.0.0
</Callout>

<Steps>

### 安装

在 Hexo 根目录执行以下命令安装主题：


<Tabs items={['npm', 'git']}>
    <Tab>
        ```bash filename="终端"
        npm install hexo-theme-redefine@latest
        ```
    </Tab>
    <Tab>
        ```bash filename="终端"
        git clone https://github.com/EvanNotFound/hexo-theme-redefine.git themes/redefine
        ```
    </Tab>
</Tabs>

### 启用主题

在 Hexo 根目录的 `_config.yml` 文件中，将 `theme` 值修改为 `redefine`。

```yaml filename="_config.yml"
theme: redefine
```

### 创建主题配置文件

在 Hexo 根目录下创建 `_config.redefine.yml` 文件。

并将[此处](https://github.com/EvanNotFound/hexo-theme-redefine/blob/main/_config.yml)的所有内容复制进去。

本文件会自动覆盖主题的配置项，创建本文件的目的是为了方便你在升级主题时，不会丢失你的配置。

### VOILA!

现在你可以启动 Hexo，看看效果了。

接下来请先阅读配置注意事项，然后继续配置主题。


</Steps>

### 注意事项

#### 1. 文档阅读说明

在接下来的配置中，`xxx.yyy` 表示你需要在 `_config.redefine.yml` 文件中的 `xxx` 下的 `yyy` 配置项中进行配置。

例如，`info.title` 指的是 `info` 下的 `title` 配置项。

```yaml filename="_config.redefine.yml"
info:
    title: Redefine
```

#### 2. YAML 语法

配置文件 `_config.redefine.yml` 使用 YAML 语法，如果你不熟悉 YAML 语法，请先阅读[此处](https://learnxinyminutes.com/docs/yaml/)。

YAML 语法是按照缩进来区分层级的，所以请注意缩进。

请保持原本的缩进，注意在配置项之前不要乱加空格。

#### 3. 配置主题

所有改动都请在 `_config.redefine.yml` 文件中进行，不建议直接修改主题文件，因为更新主题时会覆盖你的改动。
